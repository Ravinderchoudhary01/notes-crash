Topic 1: Algorithm Analysis, Asymptotic Notations & Recurrences

Algorithm analysis measures how running time and memory grow with input size. DAA focuses on machine-independent growth, not stopwatch time.

Algorithm

A finite, unambiguous sequence of steps that converts valid input into required output and terminates.

Why Analysis Matters

For large inputs, an O(n log n) design often beats an O(n^2) design even on slower hardware.

Types of Case Analysis

CaseMeaningExample
Best caseMinimum work over inputs of size n.Linear search finds the key at index 0: O(1).
Average caseExpected work over a probability distribution of inputs.Expected successful linear search: about n/2 comparisons.
Worst caseMaximum work over inputs of size n.Linear search misses the key: O(n).

Master Theorem

For T(n)=aT(n/b)+f(n):
  • If f(n)=O(n^(log_b a - epsilon)), then T(n)=Theta(n^log_b a).
  • If f(n)=Theta(n^log_b a log^k n), then T(n)=Theta(n^log_b a log^(k+1)n).
  • If f(n)=Omega(n^(log_b a + epsilon)) and regularity holds, then T(n)=Theta(f(n)).
Show recurrence tree proof for Merge Sort

T(n)=2T(n/2)+cn. At level 0 cost is cn. At level 1 there are 2 subproblems of size n/2, total cost 2*c(n/2)=cn. At level i, total combine cost remains cn. Number of levels is log2 n. Leaves contribute n*T(1)=O(n). Therefore T(n)=cn log2 n + O(n)=Theta(n log n).

Practice Questions with Solutions

Question 1:

Find the asymptotic complexity of f(n)=5n^2+30n log n+200.

Solution: The highest-order term is 5n^2. Constants and lower-order terms are ignored asymptotically, so f(n)=Theta(n^2).

Question 2:

Solve T(n)=3T(n/2)+n using Master Theorem.

Solution: Here a=3, b=2, so n^log_b a = n^log_2 3 approx n^1.585. Since f(n)=n=O(n^(1.585-epsilon)), case 1 applies. Thus T(n)=Theta(n^log_2 3).

Merge Sort

Merge Sort divides the array into halves, sorts each half recursively, and merges sorted halves in linear time. It is stable and has predictable O(n log n) time.

BestAverageWorstSpaceStable?
O(n log n)O(n log n)O(n log n)O(n)Yes
Show C Merge Sort 
#include <stdio.h>

void mergeParts(int a[], int temp[], int l, int m, int r) {
    int i = l, j = m + 1, k = l;

    while (i <= m && j <= r) {
        if (a[i] <= a[j]) temp[k++] = a[i++];
        else temp[k++] = a[j++];
    }
    while (i <= m) temp[k++] = a[i++];
    while (j <= r) temp[k++] = a[j++];

    for (i = l; i <= r; i++) {
        a[i] = temp[i];
    }
}

void mergeSort(int a[], int temp[], int l, int r) {
    if (l >= r) return;
    int m = l + (r - l) / 2;
    mergeSort(a, temp, l, m);
    mergeSort(a, temp, m + 1, r);
    mergeParts(a, temp, l, m, r);
}

Quick Sort

Quick Sort partitions the array around a pivot, then recursively sorts left and right partitions. It is usually fast in practice, but poor pivot choices cause quadratic worst case.

BestAverageWorstSpaceStable?
O(n log n)O(n log n)O(n^2)O(log n) average recursionNo
Show C Quick Sort partition logic 
void swap(int *x, int *y) {
    int t = *x;
    *x = *y;
    *y = t;
}

int partition(int a[], int low, int high) {
    int pivot = a[high];
    int i = low - 1;
    for (int j = low; j < high; j++) {
        if (a[j] <= pivot) {
            i++;
            swap(&a[i], &a[j]);
        }
    }
    swap(&a[i + 1], &a[high]);
    return i + 1;
}

void quickSort(int a[], int low, int high) {
    if (low < high) {
        int p = partition(a, low, high);
        quickSort(a, low, p - 1);
        quickSort(a, p + 1, high);
    }
}

Strassen's Matrix Multiplication

Classical matrix multiplication uses 8 recursive multiplications for 2x2 block matrices. Strassen reduces this to 7 multiplications using extra additions/subtractions.

Complexity:

Classical divide-and-conquer: T(n)=8T(n/2)+O(n^2)=O(n^3). Strassen: T(n)=7T(n/2)+O(n^2)=O(n^log2 7) approx O(n^2.807).

Show Strassen 2x2 formula set

M4=A22(B21-B11)

M6=(A21-A11)(B11+B12)
M7=(A12-A22)(B21+B22)

C11=M1+M4-M5+M7
C12=M3+M5
C21=M2+M4
C22=M1-M2+M3+M6

Practice Questions with Solutions

Question 1:

Why is Merge Sort O(n log n) in all cases?

Solution: Merge Sort always splits into two halves until single elements are reached, creating log n levels. At each level, merging all subarrays costs total O(n). Therefore total time is O(n log n) regardless of input order.

Question 2:

When does Quick Sort become O(n^2)?

Solution: If each pivot creates partitions of sizes 0 and n-1, the recurrence becomes T(n)=T(n-1)+O(n), whose sum is n+(n-1)+...+1=O(n^2).

Topic 3: Greedy Algorithms

Greedy algorithms make the locally best choice at each step. They are correct only when the problem has greedy-choice property and optimal substructure.

Exam warning: Greedy is not automatically correct. Coin change with arbitrary denominations and 0/1 knapsack are classic traps.

Fractional Knapsack

Items can be broken into fractions. Sort by value/weight ratio and take highest ratios first.

StrategyCorrectness IdeaTimeSpace
Sort by densityExchange argument: replacing lower density weight with higher density weight never hurts value.O(n log n)O(1) or sort space
Show C Fractional Knapsack
#include <stdio.h>
#include <stdlib.h>

typedef struct {
    int value;
    int weight;
} Item;

int compareDensity(const void *p, const void *q) {
    Item *a = (Item *)p;
    Item *b = (Item *)q;
    double r1 = (double)a->value / a->weight;
    double r2 = (double)b->value / b->weight;
    if (r1 < r2) return 1;
    if (r1 > r2) return -1;
    return 0;
}

double fractionalKnapsack(Item items[], int n, int capacity) {
    qsort(items, n, sizeof(Item), compareDensity);
    double answer = 0.0;

    for (int i = 0; i < n && capacity > 0; i++) {
        int take = items[i].weight < capacity ? items[i].weight : capacity;
        answer += take * ((double)items[i].value / items[i].weight);
        capacity -= take;
    }
    return answer;
}

Show C Kruskal with DSU
#include <stdlib.h>

typedef struct {
    int u, v, w;
} Edge;

int findParent(int parent[], int x) {
    if (parent[x] == x) return x;
    parent[x] = findParent(parent, parent[x]);
    return parent[x];
}

int unite(int parent[], int rank[], int a, int b) {
    a = findParent(parent, a);
    b = findParent(parent, b);
    if (a == b) return 0;
    if (rank[a] < rank[b]) {
        int t = a; a = b; b = t;
    }
    parent[b] = a;
    if (rank[a] == rank[b]) rank[a]++;
    return 1;
}

int edgeCompare(const void *p, const void *q) {
    return ((Edge *)p)->w - ((Edge *)q)->w;
}

int kruskal(int vertices, Edge edges[], int edgeCount) {
    qsort(edges, edgeCount, sizeof(Edge), edgeCompare);
    int parent[100], rank[100] = {0};
    for (int i = 0; i < vertices; i++) parent[i] = i;

    int cost = 0;
    for (int i = 0; i < edgeCount; i++) {
        if (unite(parent, rank, edges[i].u, edges[i].v)) {
            cost += edges[i].w;
        }
    }
    return cost;
}

Practice Questions with Solutions

Question 1:

For items (value, weight) = (60,10), (100,20), (120,30) and capacity 50, find the optimal fractional knapsack value.

Solution: Ratios are 6, 5, 4. Take first two fully: value 160, weight 30. Capacity left 20, take 20/30 of third item: value 80. Total 240.

Question 2:

Why does Dijkstra fail with negative edge weights?

Solution: Dijkstra finalizes the nearest unvisited vertex assuming no later path can reduce it. A negative edge discovered later can reduce a finalized distance, breaking the greedy correctness argument.

Topic 4: Dynamic Programming

Dynamic programming solves problems with overlapping subproblems and optimal substructure by storing subproblem answers.

MemoizationTabulation
Top-down recursion plus cache.Bottom-up table filling.
Computes only reached states.Often easier to optimize space and order.

0/1 Knapsack

Each item is either taken fully or skipped. State dp[i][w] = best value using first i items and capacity w.

Transition:

If wt[i-1] <= w, then dp[i][w]=max(dp[i-1][w], val[i-1]+dp[i-1][w-wt[i-1]]); otherwise dp[i][w]=dp[i-1][w].

ProblemTimeSpace
0/1 KnapsackO(nW)O(nW), optimized to O(W)

Show C 0/1 Knapsack
int max(int a, int b) {
    return a > b ? a : b;
}

int knapsack(int wt[], int val[], int n, int W) {
    int dp[101][1001];

    for (int i = 0; i <= n; i++) {
        for (int w = 0; w <= W; w++) {
            if (i == 0 || w == 0) {
                dp[i][w] = 0;
            } else if (wt[i - 1] <= w) {
                dp[i][w] = max(dp[i - 1][w],
                               val[i - 1] + dp[i - 1][w - wt[i - 1]]);
            } else {
                dp[i][w] = dp[i - 1][w];
            }
        }
    }
    return dp[n][W];
}

Longest Common Subsequence

LCS(i,j) is the longest subsequence common to prefixes X[0..i-1] and Y[0..j-1].

If characters match: dp[i][j]=1+dp[i-1][j-1]. Else: dp[i][j]=max(dp[i-1][j], dp[i][j-1]).

TimeO(mn)SpaceO(mn), reducible to O(min(m,n)) for length only.

Matrix Chain Multiplication

Given matrix dimensions, choose parenthesization minimizing scalar multiplications.

dp[i][j] = min over k in [i,j-1] of dp[i][k] + dp[k+1][j] + p[i-1]*p[k]*p[j].

TimeO(n^3)SpaceO(n^2)

Bellman-Ford

Bellman-Ford computes single-source shortest paths with negative edges and detects negative cycles reachable from the source.

RelaxationsNegative Cycle CheckTimeSpace
Relax all edges V-1 times.One more relaxation indicates a cycle.O(VE)O(V)
Show C Bellman-Ford
#include <stdio.h>

#define INF 1000000000

typedef struct {
    int u, v, w;
} Edge;

int bellmanFord(int vertices, Edge edges[], int edgeCount, int source, int dist[]) {
    for (int i = 0; i < vertices; i++) dist[i] = INF;
    dist[source] = 0;

    for (int i = 1; i <= vertices - 1; i++) {
        for (int j = 0; j < edgeCount; j++) {
            int u = edges[j].u, v = edges[j].v, w = edges[j].w;
            if (dist[u] != INF && dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
            }
        }
    }

    for (int j = 0; j < edgeCount; j++) {
        int u = edges[j].u, v = edges[j].v, w = edges[j].w;
        if (dist[u] != INF && dist[u] + w < dist[v]) {
            return 0; /* negative cycle found */
        }
    }
    return 1;
}

Practice Questions with Solutions

Question 1:

For 0/1 knapsack with weights [1,3,4], values [15,20,30], and capacity 4, what is the optimal value?

Solution: Choose item 1 and item 2: total weight 1+3=4, total value 15+20=35. Item 3 alone gives 30. Therefore the optimum is 35.

Question 2:

Why is LCS a DP problem?

Solution: It has optimal substructure because the answer for prefixes depends on smaller prefixes. It has overlapping subproblems because the same prefix pairs are reached repeatedly in recursion. Hence storing dp[i][j] avoids recomputation.

Topic 5: Backtracking & Branch and Bound

Backtracking explores a decision tree depth-first and abandons partial solutions that violate constraints. Branch and Bound also prunes using bounds on the best possible objective value.

BacktrackingBranch and Bound
Mostly constraint satisfaction: find feasible arrangements.Mostly optimization: find best-cost solution.
Prunes invalid partial states.Prunes states whose bound cannot beat current best.

N-Queens

Place N queens so no two attack each other. Place one queen per row and test columns and diagonals.

Worst TimeO(N!) with row-wise placementSpaceO(N) recursion plus board/output
Show C N-Queens
#include <stdio.h>

int board[20];

int isSafe(int row, int col) {
    for (int prev = 0; prev < row; prev++) {
        int oldCol = board[prev];
        if (oldCol == col) return 0;
        if (oldCol - prev == col - row) return 0;
        if (oldCol + prev == col + row) return 0;
    }
    return 1;
}

void printSolution(int n) {
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            printf(board[i] == j ? "Q " : ". ");
        }
        printf("\n");
    }
    printf("\n");
}

void solveNQueens(int row, int n) {
    if (row == n) {
        printSolution(n);
        return;
    }
    for (int col = 0; col < n; col++) {
        if (isSafe(row, col)) {
            board[row] = col;
            solveNQueens(row + 1, n);
        }
    }
}

Graph Coloring

Assign at most m colors to vertices so adjacent vertices have different colors. Backtracking tries colors for each vertex and rejects conflicts.

Worst TimeO(m^V)SpaceO(V)

Hamiltonian Cycle

A Hamiltonian cycle visits every vertex exactly once and returns to the start. Backtracking builds a path and checks adjacency plus no repetition.

Worst TimeO(V!)SpaceO(V)

Travelling Salesperson Problem

TSP asks for the minimum-cost tour visiting every city exactly once and returning home. Exact TSP is exponential; branch and bound prunes tours using lower bounds.

MethodIdeaTime
Brute forceTry all tours.O(n!)
DP Held-KarpState by subset and last city.O(n^2 2^n)
Branch and BoundExplore promising tours first; prune with lower bound.Exponential worst case, faster in practice.

Practice Questions with Solutions

Question 1:

In 4-Queens, why can we place at most one queen in each row?

Solution: Queens attack horizontally. If two queens are in the same row, they attack each other immediately. Backtracking therefore fixes one row at a time and only chooses a column for that row.

Question 2:

What is the key difference between pruning in backtracking and branch and bound?

Solution: Backtracking prunes infeasible partial solutions, such as a graph coloring conflict. Branch and bound prunes partial solutions whose best possible completion cannot improve the current best objective value.

P vs NP

P is contained in NP because if a problem can be solved quickly, a proposed solution can also be verified quickly. The open question is whether every quickly verifiable problem is also quickly solvable.

Current status: No proof is known for P = NP or P != NP. Most researchers believe P != NP.

Practice Questions with Solutions

Question 1:

Is every problem in P also in NP?

Solution: Yes. If a decision problem can be solved in polynomial time, then a proposed answer can also be verified in polynomial time by simply solving the problem and comparing the result.

Question 2:

How do we prove that a problem X is NP-Complete?

Solution: First show X is in NP, meaning a yes-certificate is verifiable in polynomial time. Then choose a known NP-Complete problem Y and give a polynomial-time reduction Y <=p X. This proves X is NP-Hard and therefore NP-Complete.
